## 1. PROJECT OVERVIEW & PHILOSOPHY

### 1.1 What is Firechick?

Firechick is a **multiplayer asymmetric game** (Eagle vs Chicks) that critiques how digital interfaces shape behavior, control access to knowledge, define value through measurable grades, and require multiple perspectives to reveal truth. It transforms the childhood game *麻鷹捉雞仔* (Eagle Catches Chicks) into a playable, chaotic, but reflective experience set against HKUST’s academic culture (the sundial, “GPA killers”, grade pressure).

### 1.2 Core Themes (for presentation)

- **Interface Duplication / Multi-Device Alignment** – Truth only emerges when two phones overlay visual cryptography layers (final exam).
- **Scan to Share / Restricted Knowledge Transfer** – Exam tips are scarce, transferable via QR codes, require physical proximity, and expire.
- **Measured by the System / Graded Value** – Players are reduced to letter grades, action scores, and ranked transcripts. The system tells you your worth.

**One‑line summary:** *“Scan. Share. Survive. Graduate.”*

### 1.3 Target Audience & Exhibition

- **Exhibition format:** One large host screen + multiple mobile phones (players join via QR or color code). No app installation – runs in browser.
- **Duration:** 5–7 minutes per session (fast, performative).
- **Location:** HKUST library entrance – noisy, transient, short attention span. Must be immediate, visual, and fun.

## 2. GAME MECHANICS (Complete)

### 2.1 Roles

| Role | Count (1v3) | Count (2v6) | Description |
|------|-------------|-------------|-------------|
| Eagle (GPA Killer) | 1 | 2 | Chases chicks, attacks, uses fly/cage, disrupts exam tips. |
| Chicks (Students) | 3 | 6 | Cooperate to obtain and share exam tips, survive, pass final exam. |

### 2.2 4 Game Stages

| Stage | Name | What chicks do | Eagle role |
|-------|------|----------------|-------------|
| 0 | Social Circle | Touch phones to form social circle (stage advances when all chicks have met every other chick). | Frozen (eagle awakens after 5s of game time). |
| 1 | Obtain Tips | Stand inside glowing building zones for 7s to become Star Student (get invincible prop + grade boost). | Can attack building zones (hitbox) to break protection; each hit reduces zone health by 1 (50 HP). |
| 2 | Share Tips | Share tips via QR code + proximity (must stay close for 3s to complete). | No special action. |
| 3 | Final Exam | Run to any building → visual cryptography exam (two phones overlay layers to reveal answer). | Distract chicks in real life (no in‑game action). |

### 2.3 Props & Abilities

**Chicks:**
- Speed (boost movement for 2s)
- Heal (recover 0.4 grade points)
- Teleport (target anywhere, one‑time)
- Invincible (become immune to eagle attacks for 3s)

**Eagle:**
- Attack (3s cooldown, damages chick; after valid attack: freeze eagle for 5s + attack cooldown 3s)
- Fly (10s cooldown, speed boost + ignores obstacles for 3s)
- Cage (30s cooldown, locks a random chick for 10s)

### 2.4 Minigames (Event Boxes)

Eagle can trigger a random event by touching a mystery box:

| Event | Duration | Description |
|-------|----------|-------------|
| Hitbox | 30s | Chicks and eagle tap a central button; average taps determine winner (+2/-2 grades). |
| Mock Exam | 30s | Chicks solve visual cryptography individually; correct answer gives +1 grade. |
| Crossy Road | 30s | Chicks hop lanes to cross; 2 crossings = +1 grade, 3+ = +2, <2 = -2. Eagle can speed up obstacles or add more. |

### 2.5 Grading System

- Health = letter grade (A+ to F). Starting health = 4.0 (A).
- Sub‑grade changes: +0.4 (heal), -0.4 (attack), ±2 (large grade change for events/exam).
- Action Score tracks specific actions (see formula below). Final transcript shows hidden metrics: QR scans, time in zones, tips shared, social circle completed, cooperation score.

**Action Score formulas (final):**

**Chicks:**
- Obtain tip +10
- Receive shared tip +5
- Collect prop spawn +2
- Use prop +3
- Open mystery box +10
- Crossy Road +crossings×2
- Mock Exam correct +3
- Final Exam correct +10
- Complete social circle +5
- Survival time +1 per 10s
- Hitbox tap +0.1

**Eagle:**
- Deal damage +15
- Use prop (fly) +3
- Cage chick +3
- Open mystery box +15
- Destroy building HP +0.5/tick
- Hitbox tap +0.1
- Crossy Road (eagle action) +2

### 2.6 End Game Sequence

1. After final exam or all chicks dead → **GameEndTransition** (10s animated sequence with “Pens down.”, “The exam has ended.”, “Somewhere, someone is grading…” (spinner + cycling dots), then “Your transcript is ready.”)
2. **Client:** 3‑page scroll screen (Game Over + grade + character tag → Action Score + hidden metrics → result + quote + leave)
3. **Host:** MVP showcase → winning team lineup → transcript table with score breakdown (ⓘ modal)

### 3.3 Room Code System (Immersive Mode)

- Host generates a 4‑color permutation (24 possibilities) via Neon atomic `INSERT`.
- Client join screen shows a 4‑circle picker; player taps to match the order.
- Fallback to 6‑character alphanumeric code if all permutations taken.

### 3.4 Networking: WebRTC + Cloudflare TURN

- iOS devices often fail direct peer‑to‑peer; Cloudflare TURN relays traffic.
- TURN credentials fetched from `/api/turn-credentials` (server‑side), never exposed to browser.
- Fallback to Supabase Realtime if WebRTC fails (but less reliable).

## 5. CURRENT ISSUES & BUGS (To Fix)

### 5.1 GameEndTransition (Task 1 – High Priority)

**Symptoms:**
- No loading spinner.
- “grading…” does not cycle dots.
- Lines disappear instead of staying visible together from 3s to 7s.
- Text says “PHONES DOWN” (correct, user wants to keep it).
- Transition does not appear on host or client after final exam.

**Required fixes:**
- Add `Loader2` spinner (animate-spin) next to third line.
- Cycle dots every 0.5s (grading. → grading.. → grading...).
- Keep all three lines visible from 3s to 7s (use state phase, not individual animations that clear).
- After 10s, call `onComplete` to proceed to transcript screens.

### 5.2 Exam Voting UI (Task 2 – High Priority)

**Current problem:** `ExamVotingUI` is a full‑screen modal that blocks the exam view.

**Desired behavior (inline layout):**
- Same exam screen (camera + PW layer + sliders) remains visible.
- At the bottom, show:
  - “Do you agree?” text
  - 👍 PASS / 👎 FAIL buttons (visible only after submitter submits an answer)
  - Masked answer display (e.g., “4.3” → three boxes: “4” “.” “3”)
  - Input box + SUBMIT button (only for the designated submitter; others see disabled/greyed out)
- Voting lasts 10 seconds (host‑controlled); early end if all valid voters have voted.
- Majority wins: PASS → answer accepted, exam ends; FAIL → reopen input for resubmission (same submitter).
- No separate full‑screen modal.

### 5.3 Sus Player Detection (Task 3 – High Priority)

**What works:**
- Proper quit (X button) sends `player-leave` → ⚡ icon appears in focus camera panel.

**What does NOT work:**
- Bot does **not** replace the player immediately (delayed 100ms condition in `useGameLogic.ts`).
- Ping‑pong detection (host never sends `ping` messages).
- Inactivity detection (no movement/props for 20s) does not trigger prompt.
- `AreYouStillTherePrompt` never appears.

**Required fixes in `useGameLogic.ts`:**
- Remove 100ms delay for `properQuit` – replace with bot immediately.
- Add periodic ping broadcast every 5s: `currentBroadcast({ type: "ping", ts: now })`.
- Track ping failures: if no `pong` after 2 consecutive pings, mark as sus and show prompt.
- Track last activity (movement + prop usage) per player; if idle for 20s during gameplay (not minigame/exam), trigger prompt.
- Prompt appears bottom‑right, with “Yes, I'm here” button. If no response after 15s, replace with bot.
- Sus players are still counted as “valid voters” for exam voting.
